#### _Assistant_

> 这是一个非常经典的概念混淆点。在 **0-1 背包问题**中，`dp[i][j]` 的含义是：**在前 `i` 个物品中选择，且背包容量为 `j` 时的最大价值**。
> 
> 你的直觉（上面和左面）更接近于某些路径问题（如：网格找最短路径）或**完全背包**问题，但在 0-1 背包里，逻辑是这样的：
> 
> ### 1. 为什么“上面”是对的？
> *   **上面 (`dp[i-1][j]`)**：代表容量同样是 `j`，但我**不拿**第 `i` 个物品。既然不拿，那价值自然就等于只考虑前 `i-1` 个物品时的最大价值。
> 
> ### 2. 为什么“左面”通常是错的？
> *   **左面 (`dp[i][j-1]`)**：代表容量为 `j-1` 时，考虑前 `i` 个物品的最大价值。
> *   在 0-1 背包中，我们关注的是**“拿”还是“不拿”当前的物品 `i`**。
> *   如果你取 `dp[i][j-1]`，你实际上是在说：“如果我背包容量小一点点，我的最大价值是多少”。这在填充表格时确实是一个已经算出来的值，但它**不能**代表“不拿第 `i` 个物品”的情况。
> 
> ### 3. 正确的选择逻辑
> 对于 `dp[i][j]`，只有两个选择：
> 1.  **不选物品 `i`**：价值等于 `dp[i-1][j]`（**正上方**）。
> 2.  **选物品 `i`**：价值等于 `values[i-1] + dp[i-1][j - weights[i-1]]`（**上一行的某个左侧位置**）。
> 
> ### 你的代码中 `dp[i][j-1]` 会导致什么？
> 如果你使用 `dp[i][j-1]`，你实际上是把“背包容量的变化”和“物品的选择”搞混了。
> *   0-1 背包的递推关系必须基于**“物品数量的增加”**（从 `i-1` 到 `i`）。
> *   如果你参考 `dp[i][j-1]`，由于 `dp[i][j-1]` 可能已经包含了“已经拿过物品 `i`”的情况，这会导致一个物品被**重复放入**背包（这就变成了完全背包问题的变种逻辑了）。

>     int n = nums.size();

>             {

>             }
>         }
>         // 更新全局最大值

>         }

> }

> 2.  **默认值 1**：`dp` 数组初始化为 `1`。因为每个单独的数字本身就是一个长度为 1 的递增序列。
