## 🧠 核心知识点与原理 (Core Concepts & Principles)

- **概念 1：虚拟头节点（dummy head）**
  - 通过创建 `res = new ListNode()` 作为哑节点，统一处理结果链表的头插入/尾插逻辑，避免“第一个节点要特殊处理”。
  - 最终返回 `res->next`，而不是 `res` 本身。

- **概念 2：按位加法 + 进位机制**
  - 每一轮计算：
    \[
    tmp = val1 + val2 + carry
    \]
    \[
    当前位 = tmp \% 10,\quad 新进位 = tmp / 10
    \]
  - 这是十进制逐位加法的标准流程，链表题中每个节点表示一位数字。

- **概念 3：空指针安全访问**
  - 若链表长度不同，某一方指针会先变成 `nullptr`。
  - 访问节点值时必须做保护：
    - `int val1 = curr1 ? curr1->val : 0;`
    - `int val2 = curr2 ? curr2->val : 0;`
  - 移动指针也必须先判空：
    - `if (curr1) curr1 = curr1->next;`
    - `if (curr2) curr2 = curr2->next;`

- **概念 4：循环终止条件设计**
  - 原本只写 `while (curr1 || curr2)` 会漏掉“最后还有进位”的情况。
  - 正确应为：
    - `while (curr1 || curr2 || carry)`
  - 这样当两条链都结束但 `carry == 1` 时，仍会创建最后一个节点（如 `5+5=10` 的最高位 `1`）。

## 🛠️ 关键方法与实践细节 (Key Methods & Implementation Details)

- **步骤/思路 1：结构体反注释**
  - 先恢复 `ListNode` 定义，确保编译环境具备链表节点构造函数与字段。

- **步骤/思路 2：定位原代码问题**
  - 学生代码核心问题点：
    1. 末尾无条件 `curr1 = curr1->next; curr2 = curr2->next;`，会在某一方为空时崩溃。
    2. 未处理循环结束后的剩余进位。
    3. `tmp` 计算逻辑分支多，可合并简化。

- **步骤/思路 3：采用健壮实现**
```cpp
ListNode* addTwoNumbers(ListNode* l1, ListNode* l2) {
    ListNode* res = new ListNode();
    ListNode* curr = res;
    ListNode* curr1 = l1;
    ListNode* curr2 = l2;
    int carray = 0; // 建议统一更名为 carry

    while (curr1 || curr2 || carray) {
        int val1 = curr1 ? curr1->val : 0;
        int val2 = curr2 ? curr2->val : 0;

        int tmp = val1 + val2 + carray;
        carray = tmp / 10;

        curr->next = new ListNode(tmp % 10);
        curr = curr->next;

        if (curr1) curr1 = curr1->next;
        if (curr2) curr2 = curr2->next;
    }

    return res->next;
}
```

- **步骤/思路 4：实现细节优化点**
  - 节点创建时直接赋值：`new ListNode(tmp % 10)`，少一次后续写值操作。
  - 把“取值”和“移动”都做成判空式，避免不对称链表长度导致 crash。
  - 变量名 `carray` 建议改为 `carry`（语义更清晰，减少拼写混乱）。

## ⚠️ 认知纠偏与避坑指南 (Misconceptions & Pitfalls)

- **误区/难点**：`while (curr1 || curr2)` 足够了，循环内最后统一 `curr1=curr1->next` / `curr2=curr2->next`。
- **正确认知**：只要有一个指针先为空，无条件 `->next` 就会触发空指针解引用；必须先判空再移动。

---

- **误区/难点**：只要主循环结束就可以返回，不需要额外管进位。
- **正确认知**：最高位可能仍有进位（如 `9+1` 或 `5+5`），应把 `carry` 纳入循环条件或循环后单独补节点。

---

- **误区/难点**：按三段 `if/else` 分别处理“两个都有/只有左/只有右”是必要的。
- **正确认知**：可统一用“空则按 0 处理”的方式，逻辑更短、更不易漏边界情况。

---

- **重要提示**：
  - 链表题中最常见错误不是算法思想，而是**指针边界**和**终止条件**。
  - 任何 `p = p->next` 前都要问：`p` 会不会是 `nullptr`？

## 🧠 核心知识点与原理 (Core Concepts & Principles)

- **概念 1：Dijkstra 初始化与贪心本质**
  - `dist[start]=0`：起点到自身距离为 0。
  - `pq.push({start,0})`：把起点状态入优先队列，开始扩展。
  - 每轮选取当前未确定最短路且 `dist` 最小的点（贪心正确性前提：边权非负）。

- **概念 2：`priority_queue` 比较规则**
  - `priority_queue<T>` 默认是**大顶堆**。
  - 对自定义类型 `State`，默认比较依赖 `operator<`。
  - 若要实现 Dijkstra 的“小顶堆效果”，常写：
    ```cpp
    bool operator<(const State& other) const { return dist > other.dist; }
    ```
    这是“反向定义 `<`”。

- **概念 3：运算符重载作用域**
  - `State::operator<` 只影响 `State` 类型对象比较，不影响 `int/string` 等内置类型。
  - 可重载 `<`, `==`, `[]`, `+` 等，但需语义清晰。
  - 也可不重载，改用自定义比较器（lambda/comparator）。

- **概念 4：路径恢复中的 `path[i] = start`**
  - `path[i]` 表示“到达 i 的前驱节点”。
  - 初始化时若 `start` 与 `i` 直连，则前驱设为 `start`，否则 `-1`（暂不可达）。
  - 后续松弛成功时更新：`path[v]=u`。

- **概念 5：`vis/visited` 的意义**
  - `vis[u]=true` 表示该点最短距离已“定稿”。
  - 可避免优先队列中的旧状态重复处理，提高效率与正确性。

- **概念 6：短路求值与内存安全**
  - 条件顺序必须先判边界再访问数组：
    - 正确：`curr >= 0 && s[curr] != ' '`
    - 错误：`s[curr] != ' ' && curr >= 0`
  - `&&` 有短路，`&` 没有短路，后者常导致越界。

- **概念 7：`stoi` 与进制/溢出**
  - `stoi(s)` 默认按十进制解析，不适用于二进制字符串加法题。
  - 超长字符串（如 100+ 位）转换会 `std::out_of_range`。
  - 二进制求和应使用“逐位模拟 + 进位”。

- **概念 8：前置/后置自增减**
  - `i--`：先用后减；`--i`：先减后用。
  - 在下标读取场景 `a[i--]` 常用于“读取当前位并左移指针”。

- **概念 9：整数溢出与二分**
  - `mid*mid` 可能超 `int`，触发 UB。
  - 两种稳妥写法：
    1. 用 `long long` 承接乘积；
    2. 用等价比较 `mid <= x / mid` 避免乘法。

- **概念 10：递归三要素**
  - 先判空（base case）；
  - 再访问成员；
  - 保证每个递归分支都走向终止条件（避免栈溢出）。

- **概念 11：LeetCode 注释结构体与本地环境**
  - `ListNode/TreeNode` 在 LeetCode 在线评测通常已提供，重复定义会 `redefinition`。
  - 本地 VSCode 为了补全可手动加结构体；提交前需按平台要求处理。

## 🛠️ 关键方法与实践细节 (Key Methods & Implementation Details)

- **方法 1：Dijkstra（优先队列版）关键模板**
  ```cpp
  dist[start] = 0;
  pq.push({start, 0});
  while (!pq.empty()) {
      auto cur = pq.top(); pq.pop();
      int u = cur.id;
      if (vis[u]) continue;
      vis[u] = true;
      // relax neighbors...
  }
  ```
  - 复杂度常见为 \(O(m \log n)\)。

- **方法 2：最后一个单词长度（安全写法）**
  ```cpp
  int curr = s.size() - 1, len = 0;
  while (curr >= 0 && s[curr] == ' ') curr--;
  while (curr >= 0 && s[curr] != ' ') { len++; curr--; }
  return len;
  ```

- **方法 3：二进制求和（禁止 `stoi`）**
  ```cpp
  while (i >= 0 || j >= 0 || carry) {
      int sum = carry;
      if (i >= 0) sum += a[i--] - '0';
      if (j >= 0) sum += b[j--] - '0';
      res.push_back((sum % 2) + '0');
      carry = sum / 2;
  }
  reverse(res.begin(), res.end());
  ```
  - `while` 条件含 `carry`，已自动覆盖“最高位进位”。

- **方法 4：`climbStairs` 边界与下标**
  - 若用 DP：`vector<int> dp(n+1)`，并处理 `n<=2` 直接返回。
  - 防止出现 `dp[-1]`、`dp[1]`越界（当 `n=1`）。

- **方法 5：`maxDepth` 避免重复递归**
  - 错误模式：同一子树在三元表达式里被多次递归调用。
  - 正确模式：每个子树只求一次，或直接 `return 1 + max(...)`。

- **方法 6：有序数组转 BST（108）**
  - 关键终止条件：`if (left > right) return nullptr;`
  - 不要只写 `left == right`，否则会出现区间 `left > right` 时死递归。
  - 中点建议：`mid = left + (right-left)/2`。

- **方法 7：树题空指针安全模板**
  - 先 `if (!root) ...`，再 `root->left/right/val`。
  - 这是解决 `member access within null pointer` 的第一原则。

- **方法 8：路径总和（112）正确判定**
  - 叶子判断：`!root->left && !root->right` 时比较 `target == root->val`。
  - 不应因 `target<0` 直接剪枝（节点可为负数）。

- **方法 9：最小深度（111）特判单子树**
  ```cpp
  if (!root) return 0;
  int L = minDepth(root->left), R = minDepth(root->right);
  if (L == 0) return R + 1;
  if (R == 0) return L + 1;
  return min(L, R) + 1;
  ```

- **方法 10：杨辉三角 II（119）思考框架**
  - 一维数组 `res(rowIndex+1,1)`。
  - 从后往前更新：`res[j] = res[j] + res[j-1]`，防止覆盖旧值。
  - 空间 \(O(k)\)，时间 \(O(k^2)\)。

## ⚠️ 认知纠偏与避坑指南 (Misconceptions & Pitfalls)

- **误区/难点**：`if (q = nullptr)` 也能比较空指针。  
  **正确认知**：`=` 是赋值，比较必须用 `==`。  
  **重要提示**：条件表达式里误用赋值是 C++ 常见致命 bug。

- **误区/难点**：`&` 和 `&&` 差不多。  
  **正确认知**：`&&` 才有短路保护，`&` 会继续计算右侧，可能越界。  
  **重要提示**：数组/字符串访问前必须先做下标合法性检查。

- **误区/难点**：二进制字符串可直接 `stoi` 后相加。  
  **正确认知**：默认十进制且会溢出，长输入必炸。  
  **重要提示**：大数题用字符串逐位模拟。

- **误区/难点**：树递归里可先 `root->...` 再判空。  
  **正确认知**：必须先判 `!root`。  
  **重要提示**：所有 `->` 左侧对象都要保证非空。

- **误区/难点**：`target<0` 可用于路径总和剪枝。  
  **正确认知**：节点可为负，不能这么剪。  
  **重要提示**：以“是否到达叶子”作为终止判据。

- **误区/难点**：递归边界写 `left==right` 就够了。  
  **正确认知**：区间递归必须覆盖 `left>right` 的空区间。  
  **重要提示**：否则容易无限递归 -> stack overflow。

- **误区/难点**：LeetCode 注释的 `TreeNode/ListNode` 必须反注释才能提交。  
  **正确认知**：在线环境通常已定义；本地与线上配置不同。  
  **重要提示**：本地可加定义，线上提交谨防重复定义。

- **核心议题**：厘清 C++ 函数参数在回溯函数 `backtrack(...)` 中到底是“传对象副本”还是“传地址”，并区分**值传递、引用传递、指针传递**三者。
- **涉及领域**：C++ 语言基础、函数参数传递机制、回溯算法实现细节。

## 🧠 核心知识点与原理 (Core Concepts & Principles)

- **概念 1：值传递（Pass by Value）**
  - 形参接收的是实参的一个**副本**。
  - 在函数内修改该形参，不会影响外部变量。
  - 对话中的：
    - `int start`
    - `int remaining`
    都是值传递。

- **概念 2：引用传递（Pass by Reference）**
  - 形参是实参的**别名**，语法是 `T&`。
  - 不产生对象拷贝，函数内操作会直接作用到原对象。
  - 对话中的：
    - `vector<int>& path`
    - `vector<int>& candidates`
    - `vector<vector<int>>& result`
    都是引用传递。
  - 在回溯中这很常见：避免频繁拷贝 `vector`，提升性能。

- **概念 3：指针传递（常说“传地址”）**
  - 形参写为 `T*`，调用时传 `&obj`。
  - 在函数体内需要解引用 `*p` 或 `p->...`。
  - 与引用相比更“显式地址化”，但语法更繁琐，并需考虑空指针。

- **概念 4：引用 vs 指针的关系**
  - 实践中经常说“引用本质上可看作对对象地址的间接封装”，但语言层面上两者语义不同：
    - 引用通常不可为空、不可重新绑定（初始化后绑定对象固定）。
    - 指针可以为空、可改指向。
  - 工程上：若不需要“可空/改指向”能力，优先引用更安全、可读性更好。

## 🛠️ 关键方法与实践细节 (Key Methods & Implementation Details)

- **步骤/思路 1：判断当前函数签名是何种传参**
  ```cpp
  void backtrack(int start, int remaining,
                 vector<int>& path,
                 vector<int>& candidates,
                 vector<vector<int>>& result)
  ```
  - `start`, `remaining`：值传递（复制）
  - 其余 `vector...&`：引用传递（原对象）

- **步骤/思路 2：如果要“传地址”，改成指针写法**
  ```cpp
  void backtrack(int start, int remaining,
                 vector<int>* path,
                 vector<int>* candidates,
                 vector<vector<int>>* result) {
      if (remaining == 0) {
          result->push_back(*path);
          return;
      }
  
      path->push_back((*candidates)[start]);
  }
  
  vector<vector<int>> combinationSum(vector<int>& candidates, int target) {
      vector<vector<int>> result;
      vector<int> path;
      backtrack(0, target, &path, &candidates, &result);
      return result;
  }
  ```

- **步骤/思路 3：回溯场景下的实用建议**
  - `path/result/candidates` 这类容器一般用引用传递，避免高开销拷贝。
  - 若不需要在函数内修改 `candidates`，可进一步用 `const vector<int>& candidates` 提升语义清晰度与安全性。

## ⚠️ 认知纠偏与避坑指南 (Misconceptions & Pitfalls)

- **误区/难点**：`vector<int>&` 这种写法是不是“传对象”？
- **正确认知**：它不是传副本，而是传**引用（别名）**；函数内改动会影响外部对象。

- **误区/难点**：“传引用”和“传地址”是不是完全一样？
- **正确认知**：不完全一样。引用是更安全、语法更简洁的绑定；指针是显式地址，支持空指针和改指向，灵活但更易出错。

- **重要提示**：
  - 指针参数要处理空指针风险；
  - 回溯函数中频繁调用，尽量避免大对象值传递；
  - 不修改的数据应考虑 `const T&`。
