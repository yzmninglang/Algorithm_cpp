> 在 Dijkstra 算法的初始化阶段，`path[i] = start` 的含义是：**记录当前节点 `i` 的“前驱节点”为起点 `start`**。
> 
> 具体可以从以下三个维度来理解：
> 
> ### 1. 路径回溯的原理
> Dijkstra 算法不仅要算出“最短距离是多少”，通常还要能输出“最短路径是怎么走的”。
> *   `path[i]` 存储的是：**到达节点 `i` 的前一个节点是谁**。
> *   如果我们从 `start` 直接通过一条边到达了 `i`，那么 `i` 的前一个节点自然就是 `start`。
> 
> ### 2. 代码逻辑拆解
> 这段代码通常出现在算法的**初始化步骤（Initialization）**：
> 
> ```cpp
> if (g.edge[start][i] < INF && i != start) {
>     // 如果起点 start 和节点 i 之间直接相连（边权小于无穷大）
>     // 那么目前已知的到达 i 的最短路径就是：start -> i
>     path[i] = start; 
> } else {
>     // 如果不直接相连，暂时不知道怎么去，所以标记为 -1
>     path[i] = -1; 
> }
> ```
> 
> ### 3. 后续如何使用这个 `path`？
> 算法运行结束后，如果你想知道从起点 `0` 到终点 `5` 的路径，你会像“倒着拉绳子”一样回溯：
> *   看 `path[5]`，假设是 `3`（路径最后一步是 $3 \to 5$）
> *   看 `path[3]`，假设是 `2`（路径上一步是 $2 \to 3$）
> *   看 `path[2]`，假设是 `0`（起点到了，路径上一步是 $0 \to 2$）
> *   **最终路径**：$0 \to 2 \to 3 \to 5$。
> 
> ### 关键点补充
> 在 Dijkstra 的**主循环**中，这个 `path` 还会被不断更新：
> ```cpp
> // 在松弛操作 (Relaxation) 中
> if (dist[u] + weight < dist[v]) {
>     dist[v] = dist[u] + weight;
>     path[v] = u; // 发现经过 u 去 v 更近，所以把 v 的前驱更新为 u
> }
> ```
>

#### _Assistant_

> 在算法代码中，`vis`（或 `visited`）是 **"Visited"（已访问）** 的缩写。
> 
> 它是 Dijkstra 算法中最重要的辅助数组之一，通常定义为 `vector<bool> vis` 或 `bool vis[N]`。
> 
> ### 1. 它的作用是什么？
> `vis[i]` 用来标记节点 `i` 是否已经找到了**确定的最短路径**。
> 
> *   **`vis[i] = true`**: 表示节点 `i` 已经从优先队列中弹出，并且它的最短距离已经确定，不需要再次处理。
> *   **`vis[i] = false`**: 表示节点 `i` 还在等待处理，或者虽然更新过距离，但还没确定这就是全球最优解。
> 
> ### 2. 在 Dijkstra 算法中的执行逻辑
> 在 Dijkstra 的主循环中，你会看到这样的结构：
> 
> ```cpp Student/014_dijkstra.cpp
> while (!pq.empty()) {
>     State top = pq.top();
>     pq.pop();
>     int u = top.id;
> 
>     // --- 关键判断 ---
>     if (vis[u]) continue; // 如果这个点已经访问过，跳过（因为我们已经找到过更短的路了）
>     vis[u] = true;        // 标记这个点为“已确定最短路径”
>     
>     // ... 然后去更新 u 的邻居 ...
> }
> ```
> 
> ### 3. 为什么要用 `vis`？（为了效率和正确性）
> 1.  **避免重复计算**：在 Dijkstra 的优先队列中，同一个节点可能会因为被多次“松弛”而多次入队（带着不同的距离值）。当你弹出第一个（距离最短的那个）后，其他的相同节点都是“过时”的，如果不通过 `vis` 过滤，程序会多走很多冤枉路。
> 2.  **避免环路死循环**：虽然 Dijkstra 处理带权图，但在某些逻辑下，如果没有访问标记，算法可能会在节点之间反复横跳。
> 
> ### 4. 常见的变量命名习惯
> 除了 `vis`，你可能还会在其他人的代码里看到这些名字，它们意思是一样的：
> *   `visited` (完整写法)
> *   `st` (Status 的缩写，常见于竞赛代码)
> *   `used`
> *   `found`
> 
> ### 总结
> **`vis` 就是路标上的“已完成”戳记。** 一旦 `vis[u]` 变成 `true`，就代表从起点到 `u` 的最短距离已经盖棺定论，再也不用管它了。

#### _Assistant_

> 这段代码是 **Dijkstra 算法最核心的“贪心”选择步骤**。
> 
> 在不使用优先队列（Priority Queue）的普通 Dijkstra 实现中，程序需要通过**手动遍历**来找到当前所有未确定的点中，距离起点最近的那一个。
> 
> ### 1. 逐行逻辑拆解
> 
> *   **`for (int j = 0; j < g.n; ++j)`**: 遍历图中所有的节点（从 $0$ 到 $n-1$）。
> *   **`!vis[j]`**: 这里的逻辑是：**“只看那些还没确定最短路径的点”**。如果 `vis[j]` 是 `true`，说明这个点的最短路已经找到了，不需要再看了。
> *   **`dist[j] < minDist`**: 这里的逻辑是：**“在还没确定的点里，找一个距离起点最近的”**。
>     *   `dist[j]` 是当前已知的从起点到 `j` 的距离。
>     *   `minDist` 是一个变量，用来记录目前为止发现的最小距离（初始通常设为无穷大 `INF`）。
> *   **`u = j;`**: 一旦找到了更近的点，就更新 `u`，记录下这个节点的名字。
> 
> ---
> 
> ### 2. 这个过程的标准术语：选点 (Selection)
> 
> Dijkstra 算法每一轮都要做两件事：
> 1.  **选点**：在所有“还没被标记（`!vis`）”的点里，选一个 `dist` 最小的点 `u`。这个点 `u` 的最短路径在这一刻被正式确定。
> 2.  **松弛 (Relaxation)**：用这个刚确定的点 `u` 去尝试更新它周围邻居的距离（即你代码后面会看到的逻辑）。
>

> 杨辉三角 II 的目标是返回第 `rowIndex` 行。解决这个问题的核心在于：**如何利用上一行的数据计算出当前行**。
> 
> ### 1. 核心思路：原地更新（$O(k)$ 空间复杂度）
> 
> 杨辉三角的规律是：`res[j] = res[j] + res[j-1]`。
> 为了节省空间，我们不使用二维数组，而是只维护一个长度为 `rowIndex + 1` 的一维数组。
> 
> **关键技巧：从后往前计算**
> 如果你从前往后计算（例如 `res[1] = res[1] + res[0]`），那么当你计算 `res[2]` 时，原本的 `res[1]` 已经被覆盖了。
> **从后往前更新**，可以确保你计算当前位置时，用到的前一个位置的数据依然是“上一行”的旧数据。
>

> 3.  **内层循环**：从当前行的**倒数第二个**元素开始，一直更新到**第二个**元素（索引为 1）。
