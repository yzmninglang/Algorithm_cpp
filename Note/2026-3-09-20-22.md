# C++

## 存储位置

- `std::vector` 和 `std::string` 的**对象**本身通常存储在**栈（Stack）**上（如果你是局部变量），但它们指向的存储数据的缓冲区一定是在**堆（Heap）**上。

  - **栈上的对象**：就像是一个“管理员”，它很小，只记录了指向堆的指针、当前的 `size` 和 `capacity`。主要用来存储变量名
  - **堆上的空间**：就像是“仓库”，真正的数据（如你 push_back 进去的字符或数字）都存在这里。
  - 这种“两头对跑”的设计（栈向下，堆向上）是为了**最大化利用内存空间**
  - 栈决定了递归深度，linux为8MB，win位1MB

- **堆分配比栈慢**：频繁的堆申请（频繁扩容）会拖慢程序速度。

  **内存碎片**：大量申请和释放可能导致内存碎片。

  **缓存友好性**：虽然是在堆上，但 `vector` 保证数据是**连续**的，这对 CPU 缓存非常友好，读取速度极快。



| **特性**     | **栈 (Stack)**                | **堆 (Heap)**                    |
| ------------ | ----------------------------- | -------------------------------- |
| **典型大小** | 1 MB ~ 8 MB                   | 受限于物理内存 (GB/TB 级)        |
| **分配速度** | 极快（仅需移动 CPU 指针）     | 较慢（需要在空闲列表中查找）     |
| **管理方式** | 自动（出作用域自动释放）      | 手动（需要 `delete` 或智能指针） |
| **数据地址** | 很高，向下生长                | 较低，向上生长                   |
| **溢出后果** | 程序立即崩溃 (Stack Overflow) | `new` 返回空指针或抛出异常       |

- 所有堆上的数据必然通过指针实现，所有通过栈上的数据直接赋值实现
  - 数组：int arr[100]; // 这是一个栈数组，不能太大：如果数组太大（比如 int arr[1000000]），会引发栈溢出 (Stack Overflow)。
  - 栈数组：int* p = new int[100];  //delete[] p;  **动态大小**：可以在运行时决定大小，比如 `new int[n]`。必须手动管理
  - `std::vector` 是两者的结合：对象在栈上、但是数据在堆上，上面的*p也是这样的
